objSingleTilesetAnimation

objects / Level Objects / Aesthetic / objSingleTilesetAnimation
prtAlwaysActive > objSingleTilesetAnimation

Description

An alternative to the original tile animator that animates tiles with only one background

and you can do more than one animation with one object, its recomended to keep the animated tiles in their

own layer for better load times(might with a lot of tiles + an object setter, but its a bit faster than the original tileAnimation).

How to use

Set up the variables bellow, you need to provide the top left coordinate in the background

from the region the animation starts, as well as the size of the animated region.

Note: This only uses one background per animation, for switching bagrounds use the original objTileAnimation.

Note2: the tileIDs can't be shared between animations using the same background

The following arrays must have the same size and be set in creation code

Variables

NameValueDescription
animLayer1000000

sets the layer of the animations, this object will only replace the tiles in this layer

animLayerOffset0

increase this number if two objSingleTilesetAnimation are conflicting

animateOnTransitiontrue

stopOnFlashfalse

deleteOriginalsfalse

this object makes the tiles for the animation over the originals

if for whatever reason you don't want the original non animated tiles set this to true

init0

Internal use only

variable names in bold are safe to set in creation code.
