objTileAnimationAlt

objects / Level Objects / Aesthetic / objTileAnimationAlt
prtAlwaysActive > objTileAnimationAlt

Description

An alternative to the original tile animator that lets you make animation per tile and make

more than one animation with one object, its recomended to keep the animated tiles in their

own layer for better load times(might slow with a lot of tiles + an object setter, but faster than the original tileAnimation).

How to use

Fill the variables to setup each animation and place the tiles in the room on the layer you specified in creation code

Each animation is made of a group of specific tiles, you must provide the tile index of each frame,

the tile index is the index of a tile in the background, for example: imagine you have a background

that is 4 tiles wide and 6 tiles high, and you want to use the tile at position 1,3,(positions start at 0)

then the tile id would be 1 + 4*3 = 13(position 14, but it starts at 0)

Note: This only uses one background per animation, for switching bagrounds use the original objTileAnimation.

Note2: the tileIDs can't be shared between animations using the same background

The following arrays must have the same size and be set in creation code

Variables

NameValueDescription
animLayer1000000

sets the layer of the animations, this object will only replace the tiles in this layer

init0

variable names in bold are safe to set in creation code.
